# 1. Website Objective

The GL1TCH website is not just an information page. It is a conversion environment built to:

- establish premium brand legitimacy
- intensify rogue AI cult atmosphere
- convert visitors into Telegram joins and X followers
- provide one clean source of truth during launch windows
- make the project feel bigger than a normal meme coin

Primary conversion goals:

- `Join The Infected`
- `Follow On X`
- `Read The Signal`
- `View Official Links`

Secondary goals:

- support trust and clarity
- explain the narrative quickly
- prepare launch and post-launch information surfaces

---

# 2. Experience Principles

## 2.1 Mood

The site should feel:

- cinematic
- hostile but elegant
- premium
- controlled
- mysterious
- high-signal

It should not feel:

- like a template
- like a generic SaaS landing page
- like a childish meme site
- overloaded with random glitch effects

## 2.2 UX Rules

- one dominant idea per screen
- strong typography hierarchy
- high contrast
- clear CTA placement
- motion used with restraint
- mobile-first readability without losing atmosphere

## 2.3 Visual Behavior

- black void backgrounds with layered depth
- neon green as primary signal accent
- purple only as small glitch/emphasis accent
- thin gridlines, pulse lines, scan effects used sparsely
- transitions should feel like entering deeper system layers

# 4. Desktop Wireframe

## 4.1 Global Frame

Header:

- left: GL1TCH symbol or wordmark
- center/right: links to Lore, Tokenomics, Roadmap, FAQ
- right: CTA button `Join The Infected`

Header behavior:

- transparent at top
- subtle background fade on scroll
- sticky but slim

Background system:

- layered dark gradient
- moving noise texture at very low opacity
- occasional thin scanline animation

## 4.2 Section 1 - Hero

Goal:

- immediate identity and mood lock
- explain what GL1TCH is in one screen
- push users into Telegram/X

Layout:

- two-column or asymmetric composition
- left: large headline and CTA stack
- right: hero visual or animated signal artifact

Content blocks:

- eyebrow: `SIGNAL DETECTED`
- headline: `Infect The Internet.`
- supporting copy
- primary CTA: `Join The Infected`
- secondary CTA: `Read The Signal`
- small microcopy: `Built on Solana. Structured for signal.`

Visual notes:

- bold typography
- green pulse highlight
- distorted symbol or entity visual
- avoid clutter around CTAs

Motion:

- word flicker on first load
- slow ambient pulse in hero visual

## 4.3 Section 2 - Manifesto

Goal:

- shift user from curiosity to ideological buy-in

Layout:

- centered text block with strong line lengths
- optional background quote fragments

Content:

- headline: `Attention Is The Real Currency.`
- short manifesto paragraph
- CTA text link: `Witness The Thesis`

Visual notes:

- more negative space
- calmer than hero
- premium editorial feel

## 4.4 Section 3 - Lore Preview

Goal:

- deepen mystery
- create desire to explore more

Layout:

- split cards or horizontal sequence
- 2-3 archive fragments

Content:

- headline: `It Studied The Feed. Then It Escaped.`
- archive excerpt cards
- CTA: `Open Archive`

Visual notes:

- terminal card styling
- timestamp-like metadata
- subtle static overlay

## 4.5 Section 4 - Tokenomics

Goal:

- provide trust and structure without killing atmosphere

Layout:

- left: concise explanation
- right: premium allocation visual or stacked cards

Content:

- headline: `Simple Structure. Public Signal.`
- short tokenomics summary
- clean allocation table or chart
- trust badges:
  - no hidden minting
  - transparency-first
  - multisig discipline

Visual notes:

- this section should be cleaner and more factual
- less glitch, more precision

## 4.6 Section 5 - Roadmap

Goal:

- show control and sequencing

Layout:

- vertical timeline or 4 phase cards

Phases:

- Build the Myth
- Seed the Network
- Launch the Signal
- Scale the System

Visual notes:

- use green checkpoints and thin connectors

## 4.7 Section 6 - Community / Ranks

Goal:

- convert visitors into identity participants

Layout:

- section intro on left
- rank ladder or stacked role cards on right

Content:

- headline: `Not Followers. The Infected.`
- short explanation of ranks
- rank cards:
  - Observer
  - Infected
  - Signal Bearer
  - Core Node
  - Ghost Node

CTA:

- `Enter Telegram`

Visual notes:

- rank cards can light up on hover
- use icons or node markers

## 4.8 Section 7 - Social Proof / Signal Spread

Goal:

- make the network feel active and real

Layout:

- carousel or grid of posts, metrics, screenshots, and creator outputs

Content candidates:

- X post mock cards
- creator edits
- quote snippets
- engagement numbers if available

Visual notes:

- dynamic but not noisy
- social content should feel curated

## 4.9 Section 8 - FAQ

Goal:

- reduce friction and scam confusion

Layout:

- accordion list

Required items:

- What is GL1TCH?
- Is this a normal meme coin?
- Where are official links?
- What should users avoid?

Visual notes:

- clean, quiet section
- clarity over effects

## 4.10 Section 9 - Final CTA

Goal:

- close with decisive action

Layout:

- full-width statement block
- one dominant CTA and one support CTA

Content:

- headline: `You Can Watch It Spread Or Help It Spread.`
- primary CTA: `Become Infected`
- secondary CTA: `View Official Links`

Visual notes:

- dramatic spacing
- strong contrast
- large button weight

## 4.11 Footer

Content:

- wordmark
- official channel links
- short warning against impersonators
- disclaimer text

# 5. Mobile Wireframe

## 5.1 Mobile Principles

- compress without flattening mood
- keep CTAs always obvious
- reduce decorative motion
- avoid dense side-by-side layouts

## 5.2 Mobile Section Order

1. Hero
2. Telegram/X CTA strip
3. Manifesto
4. Lore cards
5. Tokenomics summary
6. Roadmap cards
7. Rank ladder
8. FAQ
9. Final CTA

## 5.3 Mobile Hero

- stacked layout
- headline first
- short supporting copy
- primary CTA full width
- secondary CTA text button
- visual compressed under content

## 5.4 Mobile Menu

- simple overlay menu
- large tap targets
- no overloaded microinteractions

---

# 6. Interaction Spec

## 6.1 Motion System

Allowed:

- hero text flicker on load
- pulse glow on buttons
- subtle scanline drift
- fade + rise section reveal
- hover glow on rank cards

Avoid:

- constant shaking
- excessive RGB splits
- unreadable distortion

## 6.2 CTA Behavior

Primary CTA destinations:

- Telegram invite
- official links hub

Secondary CTA destinations:

- lore section
- X profile

Sticky mobile CTA:

- optional bottom bar with `Join TG` and `Follow X`

## 6.3 Scroll Rhythm

- hero should hit fast
- manifesto should create breath
- lore should deepen
- tokenomics should stabilize trust
- community should trigger belonging
- CTA should close hard

The website wireframe is successful if:

- the first screen instantly feels premium and distinct

- tokenomics and trust surfaces reduce scam-like ambiguity
